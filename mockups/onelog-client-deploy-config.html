① LOG SERVER (1 node)

Máy chạy toàn bộ stack Docker Compose. Repo clone về ~/onelog.

  • Chạy: Vector · VictoriaLogs · Qdrant · Postgres · Redis · NATS
  • App: Agent · Indexer · MCP-VL · MCP-Semantic
  • Chat: LiteLLM · OpenWebUI · mcpo · Caddy · Alertmanager · vmalert
  • Ports mở LAN: 514/udp 6514/tcp 6515/tcp 80 443
② CLIENT SERVERS (N nodes)

Mỗi client chỉ cần rsyslog forward tới log server. Không cần Docker.

  • 1 file duy nhất: /etc/rsyslog.d/90-forward-onelog.conf
  • Forward TCP 6514 (RFC5424) — có disk queue khi log server down
  • Setup 1 lệnh: setup-rsyslog-client.sh
  • Optional: Vector agent (buffer đĩa + TLS-ready) trong infra/clients/

905 GB /opt/ragstack/data đủ cho 1000 clients @ 30d retention.
Probe topology — 2 nguồn: Vector exec probe data disk (bind /opt/ragstack/data:/host/data:ro,rslave) + host cron probe / curl VL trực tiếp. KHÔNG bind rootfs (bảo mật).
Anti-injection — Vector transform hard-set source_stream:vector-exec-probe. LogsQL rule filter theo cả service: AND source_stream: → attacker container log không thể forge event suppress alert.
Alert latency — Critical (for:0s) 5-10 phút. Warning (for:15m) 15-25 phút. Probe cadence 5m + vmalert eval 5m + Alertmanager group_wait 30s.

# Client KHÔNG cần Docker · KHÔNG clone repo
# Chỉ cần rsyslog (đa số distro có sẵn)

/etc/
├── rsyslog.conf                # main config (thường không sửa)
├── rsyslog.d/                  # ★ drop-in configs
│   ├── 90-forward-onelog.conf   # ★★ FILE CHÍNH — forward TCP 6514
│   ├── 91-forward-onelog-json.conf # (opt) JSON ECS-lite TCP 6515
│   ├── *.bak                    # config cũ script tự backup
│   └── 50-default.conf          # local /var/log/* (giữ nguyên)
├── hosts                        # (opt) map logserver IP → tên
├── hostname                     # tên host — hiện trong VL field `host`
├── timezone                     # ★ đồng bộ với logserver (Asia/Saigon)
├── chrony/chrony.conf           # ★ NTP — timestamp lệch = trace vỡ
└── ufw/ hoặc iptables/       # firewall outbound rule

/var/
├── log/                         # log gốc client (rsyslog đọc từ đây)
│   ├── syslog · auth.log · kern.log
│   └── <app>/                     # app log dạng file (nếu có)
└── spool/rsyslog/              # ★ disk queue khi logserver down
    └── onelog_fwd.*.qi/.qf       # queue file (500 MB cap)

/lib/systemd/system/
└── rsyslog.service             # unit — enable on boot

# Nếu chạy Vector agent thay rsyslog (option nâng cao):
/etc/vector/
└── vector.yaml                 # sources file/journald → sink TCP
/var/lib/vector/                  # disk buffer
Chi tiết từng file
File Vai trò · Khi nào sửa
/etc/rsyslog.d/
90-forward-onelog.conf		 MAIN Forward tất cả log → logserver TCP 6514.
Sửa khi: đổi IP logserver, đổi port, chuyển protocol, chỉnh queue size, đổi template RFC5424.
/etc/rsyslog.d/
91-forward-onelog-json.conf		 OPT Format JSON ECS-lite qua TCP 6515.
Dùng khi client đã có pipeline JSON (ELK/SIEM) và muốn OneLog làm sink phụ.
/etc/rsyslog.conf Main config rsyslog. Thường KHÔNG sửa — mọi tuỳ biến đưa vào rsyslog.d/*.conf.
/etc/hostname Tên host xuất hiện ở field host trong VictoriaLogs.
Đặt tên gọn, unique (srv-01, srv-02, api-prod-1…).
/etc/timezone
/etc/chrony/chrony.conf		 BẮT BUỘC NTP sync trước khi ingest.
timedatectl set-timezone Asia/Saigon + systemctl enable --now chrony. Timestamp lệch → trace vỡ, redaction sai.
/etc/hosts OPT Map logserver → IP để config đọc dễ.
Sửa khi lab DNS chưa có.
/etc/ufw/ · iptables Firewall outbound. Cho phép TCP 6514 (hoặc 6515) → logserver IP. Đa số client không chặn outbound, nhưng nên xác nhận.
/var/spool/rsyslog/
onelog_fwd.*		 AUTO Disk queue rsyslog tự quản lý (max 500 MB).
Buffer khi logserver down — resume tự động. KHÔNG xoá tay khi rsyslog đang chạy.
/var/log/*.log Log gốc — rsyslog vẫn ghi local song song với forward.
Giữ logrotate mặc định để không đầy disk.
/etc/systemd/journald.conf OPT Nếu app dùng journald, đảm bảo ForwardToSyslog=yes để rsyslog thấy được.
/etc/vector/vector.yaml OPT-adv Nếu chạy Vector agent thay rsyslog (buffer đĩa + TLS-ready).
Template trong repo: infra/clients/ (bên logserver).
Kịch bản chỉnh sửa trên client
Cần làm Sửa file Nội dung · Lệnh reload
Đổi IP log server /etc/rsyslog.d/90-forward-onelog.conf target="<IP mới>" → systemctl restart rsyslog
Đổi port (VD chuyển TLS 6514→6515) …90-forward-onelog.conf port="6515" + nếu TLS thì cấu hình StreamDriver
Tăng disk queue (client log nhiều) …90-forward-onelog.conf queue.maxDiskSpace="2g" · queue.size="50000"
Chỉ forward log của 1 app …90-forward-onelog.conf Thay *.* bằng if $programname == 'myapp' then …
Bỏ forward (rollback) …90-forward-onelog.conf rm file → systemctl restart rsyslog
Đổi hostname hiển thị trong VL /etc/hostname hostnamectl set-hostname <name> → restart rsyslog
Sync giờ (lệch time zone) /etc/chrony/chrony.conf
/etc/timezone		 timedatectl set-timezone Asia/Saigon · systemctl restart chrony
Mở outbound firewall ufw / iptables / firewalld ufw allow out to <logserver> port 6514 proto tcp
Chuyển sang JSON ECS-lite /etc/rsyslog.d/91-forward-onelog-json.conf Copy từ infra/clients/rsyslog-forward-json.conf, replace IP, port 6515
Xem queue có backlog không /var/spool/rsyslog/ ls -lh /var/spool/rsyslog/onelog_fwd.* — nhiều file = logserver đang down
Debug: log không tới server journalctl -u rsyslog ss -tnp | grep :6514 phải ESTAB · rsyslogd -N1 check syntax
Verify trên client
# 1. Rsyslog đang chạy systemctl is-active rsyslog # 2. TCP tới logserver ESTAB ss -tnp | grep :6514 # 3. Config OK sudo rsyslogd -N1 # 4. Fire smoke log logger -t hello-onelog "from $(hostname)" # 5. Bên logserver xác nhận curl -s 'http://<logserver>:9428/select/logsql/query' \ --data-urlencode 'query=hello-onelog' | jq .
Rollback / gỡ khỏi client
sudo rm /etc/rsyslog.d/90-forward-onelog.conf sudo systemctl restart rsyslog # (opt) khôi phục config cũ nếu setup script backup sudo ls /etc/rsyslog.d/*.bak sudo mv /etc/rsyslog.d/<file>.bak /etc/rsyslog.d/<file> # Nếu dùng Vector agent sudo systemctl disable --now vector sudo rm -rf /etc/vector /var/lib/vector
TL;DR — Client chỉ cần 1 file: /etc/rsyslog.d/90-forward-onelog.conf. Mọi thứ khác là bonus/verify.
Prereq bắt buộc — NTP sync (chrony) + timezone khớp logserver. Timestamp lệch = log vô nghĩa.
Không cần — Docker, Python, systemd unit riêng cho OneLog. Không clone repo về client.
Quick deploy — thứ tự copy/paste
LOG SERVER
# 1. Base + NTP + clone sudo apt-get install -y curl git ca-certificates ufw chrony git clone <repo> ~/onelog # 2. One-shot setup (Docker + UFW + hosts) sudo INVOKING_USER=$USER \ LAN_CIDR=192.168.122.0/24 \ bash ~/onelog/infra/scripts/setup-log-server.sh # 3. Secrets → .env cd ~/onelog/infra cp .env.example .env openssl rand -hex 24 # paste vào POSTGRES_/QDRANT_/REDIS_ vi .env # 3b. LLM keys (nếu dùng LiteLLM proxy) cp litellm/.env.llm.example litellm/.env.llm chmod 0400 litellm/.env.llm vi litellm/.env.llm # 4. Up full stack docker compose \ --profile agent --profile mcp \ --profile alerts --profile indexer \ --profile chat --profile llm \ up -d # 5. Systemd autostart sudo bash infra/scripts/install-systemd-unit.sh # 6. Snapshot daily crontab -e # 0 2 * * * scripts/snapshot-daily.sh
CLIENT (mỗi host)
# 1 lệnh duy nhất sudo LOG_SERVER_IP=<logserver-ip> \ bash /tmp/setup-rsyslog-client.sh # Verify ss -tnp | grep :6514 logger -t hello "from $(hostname)" # Trên log server check curl -s 'http://localhost:9428/select/logsql/query' \ --data-urlencode 'query=hello'
SMOKE TEST
bash ~/onelog/infra/scripts/healthcheck.sh docker compose ps curl -fsS http://localhost:9428/health curl -N -X POST http://localhost:8080/chat \ -H 'Content-Type: application/json' \ -d '{"query":"lỗi connection reset?"}'
