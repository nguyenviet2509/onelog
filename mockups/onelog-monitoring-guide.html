Pipeline Health — sức khoẻ đường log

refresh 30s Last 15m khuyến nghị

Xem 1 lượt hàng sáng để chắc pipeline log vẫn chạy. Bất kỳ panel nào "No data" hoặc trend bất thường → điều tra.

Ingest rate (VL, bytes/s)

rate(vl_bytes_ingested_total[1m])

Tốc độ log Vector đẩy vào VictoriaLogs. Zero → syslog client mất kết nối hoặc Vector chết. Spike đột biến → loop log hoặc app đổ debug.

bình thường 0.5–5 MB/s >20 MB/s: check spam = 0 > 5 phút

Indexer NATS lag (seconds) + batches/s

indexer_ingest_lag_seconds · rate(indexer_batches_processed_total[1m])

Lag = giây log nằm trong NATS chưa được embed. batches/s = throughput indexer. Lag tăng đều → embed chậm (OpenAI rate limit, mạng chậm) hoặc load quá cao.

lag < 30s 30–120s: watch > 300s: escalate

Qdrant collection points (log_templates)

collection_points{id="log_templates"}

Số template log đã cluster + embed. Tăng đều = indexer hoạt động. Chững lại vài giờ = indexer treo hoặc embed API 4xx/5xx.

tăng theo giờ phẳng 30 phút giảm bất thường

VL storage (GB)

vl_data_size_bytes / 1024^3

Dung lượng VictoriaLogs. Tăng đều theo retention (mặc định 7d). Nếu tăng vọt → app đổ log lỗi lặp.

tăng chậm, ổn định sau 7d gấp 2× baseline/ngày

Component up

up{job=~"onelog-.*"}

Trạng thái scrape /metrics của mỗi service. DOWN trong > 2 phút = container hoặc endpoint có vấn đề. Xoá series stale nếu target đã đổi.

tất cả UP bất kỳ DOWN

Recent alerts (vmalert)

ALERTS{alertstate="firing"}

Alert đang firing. Rỗng = OK. Có alert → đọc labels (severity, service, mount) để hành động; Telegram cũng đã push song song.

bảng rỗng severity=critical

Host sparkline (CPU / RAM / net)

3 stat panels, click drill-down → VPS Host

Cover VPS-level. Đỏ ở đây trước khi pipeline bị ảnh hưởng → xử lý host trước.

CPU < 70% · RAM < 75% 70–85% > 90%

VPS Host — CPU / RAM / mạng / đĩa

refresh 30s Last 1h khuyến nghị

Khi Pipeline Health báo bất thường mà không rõ nguyên nhân, mở dashboard này. Cũng dùng cho capacity planning (khi nào cần scale VPS).

CPU % (by mode)

avg by (mode) (rate(node_cpu_seconds_total[1m])) * 100

Chú ý màu iowait (chờ disk) và softirq (network irq overload). user + system nhiều = compute-bound.

idle > 40% iowait > 20% idle < 10% liên tục

Load average (1/5/15m)

node_load1 · node_load5 · node_load15

Số process ready-to-run. So với số core (VPS 8 cores). load1 > core count → đang queue.

< 8 (số core) 8–12 > 16 kéo dài

Memory (used / cached / free)

MemTotal − MemFree − Buffers − Cached · Cached+Buffers · MemFree

Cached lớn = OK (Linux dùng RAM làm disk cache). Chỉ lo khi used gần bằng total và free + cached nhỏ.

used < 75% 75–90% > 90% OOM risk

Swap used

SwapTotal − SwapFree

VPS hiện KHÔNG có swap (0B). Nếu số > 0 => ai đó bật swap thủ công. RAM đang thiếu → xem Memory panel.

= 0 > 0

Root disk used %

100 − (avail/size × 100), mountpoint="/"

Đĩa 100 GB. VL retention 7d nên baseline < 20 GB. Vọt > 80% → check docker system df, snapshot backups đầy.

< 70% 70–85% > 85% (vmalert fires)

Network throughput (rx / tx bytes/s)

rate(node_network_{receive,transmit}_bytes_total[1m]), interface không phải docker/veth

Đo card mạng thật. Rx spike bất thường → log flood từ client. Tx spike → LLM proxy trả nhiều token hoặc backup S3 upload.

< 10 MB/s baseline sustained > 50 MB/s

Disk I/O bytes/s (read / write)

rate(node_disk_{read,written}_bytes_total[1m])

Write spike thường do VL flush index hoặc backup snapshot. Read spike thường do VMUI truy vấn dài. iowait CPU panel là chỉ báo sớm hơn.

< 100 MB/s peak > 200 MB/s liên tục

Alert flow — từ metric sang Telegram

VictoriaLogs / VictoriaMetrics
        │  (metric / LogsQL query)
        ▼
    vmalert       # evaluates rules mỗi 15s
        │  (fires ALERTS{alertstate="firing"})
        ▼
    alertmanager  # dedupe + group + route
        │
        ▼
    Telegram bot → group chat (thread_id theo severity)

Rules ở infra/vmalert/rules.yml. Routes ở infra/alertmanager/config.yml. Test cơ chế alert:

# fake alert xuyên qua toàn bộ chain docker exec ragstack-alertmanager amtool alert add TestAlert \ severity=critical service=api-1 --annotation="summary=Smoke test"

Playbook triage 5 phút

  1. Mở Pipeline Health · time range Last 15m.
  2. Check Component up — bất kỳ DOWN → nhớ tên service.
  3. Check Ingest rate khác 0 và tương đương baseline hằng ngày.
  4. Check Indexer lag < 30s. Nếu tăng nhanh → embed API có vấn đề.
  5. Sparkline CPU/RAM/net — nếu đỏ, mở VPS Host.
  6. Cuối cùng liếc Recent alerts (vmalert) — nếu có, đọc labels và annotate summary.

Nếu tất cả xanh mà user vẫn báo lỗi → vấn đề nằm ngoài telemetry (DNS, LAN, browser). Kiểm tra client-side / Caddy access log.
