Fan-out tại vector

1 line log → ghi VictoriaLogs (full, query được). Chỉ WARN+ mới publish NATS → indexer → Qdrant. Tiết kiệm chi phí embedding ~100×.

Xóa PII trước khi lưu

Vector VRL redact email · RFC1918 IP · JWT · AWS key · Bearer · password=… trước khi ghi. Downstream luôn đọc data sạch.

Nhấn khi trình bày: chỉ tay theo mũi tên ingest — App → vector → VL + NATS → Qdrant. Đừng đọc từng service.

6 Pipeline cảnh báo · 3 phút · Từ log tới Telegram
sequenceDiagram autonumber participant V as vector participant VL as VictoriaLogs participant VA as vmalert participant AM as alertmanager participant TG as Telegram V->>VL: insert log line loop 30s instant / 5m burst VA->>VL: LogsQL stats_query (rule expr) VL-->>VA: series {host, host_ip, sample_msg} alt expr trả row VA->>AM: firing else expr rỗng VA->>AM: resolved end end AM->>AM: dedupe · group_by(alertname,host) AM->>TG: sendMessage (HTML template)
Instant rules (eval 30s)

OomKill · DiskFull · NewUserOrSudoer. Fire NGAY khi 1 event match — không chờ.

Burst rules (eval 5m, for=1m)

SSH brute force · MySQL err burst · kernel err. Cần >N event/5m mới fire.

Show alert thật: nếu có Telegram history sẵn, mở ra cho đồng nghiệp thấy format thực tế (host_ip, sample_msg, started/ended giờ VN).

Log có bị lộ lên LLM không?↕

Chỉ evidence ngắn (đoạn log liên quan) được gửi lên LLM. Toàn bộ log gốc ở lại on-prem trên VictoriaLogs. Trước khi lưu, vector đã redact PII/secret (email · IP · JWT · AWS key · Bearer · password=). 2 lớp bảo vệ.

Mất internet — OneLog còn dùng được không?↕

Ingest + LogsQL query nội bộ vẫn chạy. Nhưng LLM cần internet → mất internet thì không hỏi natural language được. Fallback: vào VMUI :9428 query LogsQL tay.

Scale lên 30 VM thì đổi gì?↕

Xem docs/ha-roadmap.md. Tóm tắt: vector multi-node · VictoriaLogs cluster mode · Qdrant replica · NATS jetstream cluster · Caddy → HAProxy/Nginx LB. Không cần rewrite, chỉ nhân bản node.
