OneLog là gì

Một "log brain" duy nhất cho 5 ops — hỏi tiếng Việt, ra log production kèm citation.

OneLog gom log từ tất cả VM về một VictoriaLogs trung tâm, đồng thời index semantic các template log vào Qdrant. Ops mở Claude Desktop, hỏi "mysql có lỗi gì 1 giờ qua không?" — Claude tự chọn tool MCP phù hợp (mcp-vl precise LogsQL hoặc mcp-semantic fuzzy template search), trả về trích dẫn [service:host:ts] + URL VMUI để bấm xem raw log. Không còn web UI — toàn bộ UX nằm trong Claude Desktop.

3  Pipeline ghi log — từ VM đến lưu trữ

sequenceDiagram autonumber participant App as App on srv-01/02 participant Rs as rsyslog/vector-agent participant V as vector (logserver) participant VL as VictoriaLogs participant N as NATS participant I as indexer participant Q as Qdrant App->>Rs: stdout / /var/log/* Rs->>V: syslog UDP 514 / TCP 6514 / JSON 6515 V->>V: enrich (service,host,severity) → redact PII/secrets par V->>VL: ES bulk JSON insert (all events) and V->>V: filter severity ∈ {warn,err,crit,alert,emerg} V->>N: publish "logs.warn" (WARN+ only) end N->>I: JetStream pull-consume I->>I: extract template (drain3) + embed (small-model) I->>Q: upsert vector + payload{template,service,host,first_seen,last_seen,count} Note over Q: Qdrant payload là
source of truth template meta
Điểm nổi bật
  • • Fan-out tại vector: 1 line log → ghi VictoriaLogs (full, có thể query). Chỉ WARN+ mới publish lên NATS logs.warn (giảm chi phí embedding).
  • • PII/secret redaction trước khi lưu: regex VRL replace email, RFC1918 IP, JWT, AWS key, Bearer token, password=... → downstream luôn đọc data sạch.
  • • Template extraction (drain3): gom các line giống nhau về 1 "template" (ví dụ Connection refused to <host>:<port>) → giảm 100× lượng embedding.
  • • NTP bắt buộc: chronyd 3 VM, lệch >1s sẽ phá trace.
  • • Backpressure: nếu Qdrant chậm, NATS JetStream giữ queue; vector vẫn ghi VL bình thường — query precise không bị block.

Câu hỏi mơ hồ

"vì sao", "có vấn đề gì", "bất thường"... → mcp-semantic. Embed → Qdrant top-k.

Lọc chính xác

"service=nginx AND status=502 AND _time:1h" → mcp-vl. LogsQL trực tiếp.

Hợp đồng citation

Mọi câu trả lời PHẢI có [service:host:ts] + vmui_url. Không cite = không tin.

5  Các MCP tool Claude có thể gọi

onelog-vl precise
/mcp/vl/sse

Server mcp-victorialogs chính chủ — không tự code, expose toàn bộ LogsQL tools.

  • query(logsql, time_range) — LogsQL thô, trả về các dòng log
  • hits(filter, top_n) — top N theo field
  • stats_query(group_by, agg) — đếm/percentile
  • vmui_url(query) — sinh deeplink VMUI
# ví dụ
hits(filter="service:nginx status:5*",
     range="_time:30m", top_n=10)
onelog-semantic fuzzy
/mcp/semantic/sse

FastMCP custom · 1 tool duy nhất theo thiết kế (KISS) · Streamable HTTP transport.

  • search_log_templates(query, filters?, top_k?) — embed → Qdrant top-k → trả về template + sample line + vmui_url để drill xuống mcp-vl
# Claude tự chain:
1. search_log_templates(
     query="mysql connection pool exhausted",
     filters={"service":"mysql"})
   → trả về template_id + vmui_url

2. mcp-vl.query(logsql=vmui_url.logsql)
   → raw lines kèm citation

Nhóm rule · routing
  • • instant (eval 30s, for=0s): OomKill / DiskFull / NewUserOrSudoer — fire ngay khi 1 sự kiện match.
  • • burst (eval 5m, for=1m): SSH brute force, MySQL err burst, kernel err... — cần >N event/5m.
  • • Route: alert có label notify_style=event → receiver telegram-event (send_resolved=false); còn lại → telegram-trend (send_resolved=true).
  • • Resolve data-driven: RESOLVED chỉ về khi LogsQL không còn trả series cho host đó. Không có timer cố định.
Lỗi thường gặp
  • • Đổi alertmanager.yml: entrypoint sed render vào /tmp/ chỉ tại start → phải --force-recreate, reload không pick file mới.
  • • Timestamp UTC: container thiếu tzdata → mount /usr/share/zoneinfo + set TZ=Asia/Ho_Chi_Minh.
  • • Facility remap: logger -p kern.* bị Linux đổi thành user → rule expr match theo service: hoặc msg pattern, không filter facility:.

FAQ nhanh

[UPDATED 2026-07] Vì sao restore agent + thêm OpenWebUI?↕

Plan 260701-1544 (LLM Provider Abstraction): mở đường thay 4 provider (Claude/GPT/Gemini/DeepSeek) qua 1 env var để giảm cost (Gemini Flash raw ~12% Claude). Agent restored với LiteLLM adapter thay Anthropic direct SDK — API /chat SSE dùng lại cho integration ngoài. OpenWebUI cho team ops muốn UI web thay Claude Desktop (không phải bắt buộc migrate). Cả 2 UX song song, chọn 1 trong 2 hoặc cả 2.

Vì sao trước đây bỏ web UI + rag-agent? Chỉ dùng MCP qua Claude Desktop?↕

Phase 02 (2026-06-24): decommission Next.js web + FastAPI agent (Anthropic direct SDK) vì duplicate cost với Claude Team subscription. Chỉ giữ 2 MCP server thin. Đến 2026-07, agent restored vì có LiteLLM abstraction — không còn hard-code 1 provider, có thể dùng key rẻ hơn. Web UI cũ vẫn commented, thay bằng OpenWebUI (off-the-shelf).

Tách 2 MCP server (vl + semantic) thay vì gộp 1?↕

Để Claude tự routing theo intent. mcp-vl chạy LogsQL trực tiếp (nhanh, rẻ, không embed). mcp-semantic phải embed query (chậm hơn, có cost). Tách giúp audit + rate-limit theo loại, và mcp-vl tận dụng upstream mcp-victorialogs không phải code.

Resurrect web UI khi cần?↕

Uncomment 2 block web + agent trong infra/docker-compose.yml, set ANTHROPIC_API_KEY trong .env, chạy docker compose --profile web --profile agent up -d. Drill: infra/scripts/resurrect-drill.sh. Branch legacy-web giữ snapshot bootable.

Vì sao Qdrant lưu template chứ không phải từng log line?↕

Một service đẩy ~10⁶ line/ngày nhưng chỉ ~10² template. Embed line-by-line = đốt cost & nhiễu vector space. Template (drain3) gom các line cùng pattern → semantic chính xác hơn.

Mất internet — OneLog còn dùng được không?↕

Ingest + LogsQL query nội bộ vẫn chạy. Nhưng Claude Desktop cần Anthropic API → mất internet thì không hỏi natural language được. Fallback: vào VMUI :9428 query LogsQL tay.

Scale lên 30 VM thì kiến trúc đổi gì?↕

Xem docs/ha-roadmap.md. Tóm tắt: vector multi-node, VictoriaLogs cluster mode, Qdrant replica, NATS jetstream cluster, Caddy → HAProxy/Nginx LB.
