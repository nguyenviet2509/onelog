Discovery — máy nào, service nào đang gửi log

Tất cả host đang gửi log (5 phút)
Inventory nhanh — biết đủ số client server không.
* | stats by (host) count()
Tất cả service theo count
Xem service nào đang chảy nhiều nhất — spot rare/silent service.
* | stats by (service) count() as n | sort by (n desc) | limit 30
Ma trận host × service
Bảng tổng: mỗi host gửi service nào bao nhiêu. Dùng để phát hiện service thiếu.
* | stats by (host, service) count() | sort by (count() desc) | limit 100
Host im lặng — kiểm tra client mất kết nối
Nếu bạn kỳ vọng có N client → xem con số này có đủ không. Số 0 = client đó chết.
* | stats by (host) count() as events | sort by (events)

Errors & Critical — cái gì đang cháy

Mọi log err+ (15 phút gần nhất)
Query mở màn triage — nhìn 1 lượt các event nặng.
severity:(err OR crit OR alert OR emerg)
Error rate theo service (1h)
Xếp hạng service nào đang ném error nhiều. Top là ứng viên điều tra.
severity:err | stats by (service, host) count() as errors | sort by (errors desc) | limit 20
Top error message
Message nào lặp nhiều nhất → cụm bug cần fix ưu tiên.
severity:err | stats by (_msg) count() as n | sort by (n desc) | limit 20
Panic / Fatal / Segfault
Sự kiện app chết bất thường — cần root cause ngay.
_msg:(panic OR fatal OR segfault OR "core dumped")
Connection refused / timeout
Bắt cả 2 pattern phổ biến của outage giữa các service.
_msg:("connection refused" OR "connection timed out" OR "no route to host")
Restart loop detector
Service systemd bị restart liên tục → thường là crash + auto-restart. > 3 lần trong 5m là bất thường.
_msg:"Started" | stats by (service, host) count() as restarts | filter restarts:>3

Security & Auth

SSH login FAIL
Bruteforce detection. Nhìn tần suất theo user + src IP.
service:sshd _msg:("Failed password" OR "Invalid user")
SSH login SUCCESS
Xem ai vừa login vào host nào. Correlation lúc có sự cố.
service:sshd _msg:"Accepted"
Sudo events — chạy quyền root
Ai chạy sudo, lệnh gì. Audit trail cơ bản.
service:sudo
User account creation / mod
Bất kỳ event tạo/sửa user, group → có alert riêng nhưng vẫn nên spot check.
service:(useradd OR usermod OR groupadd OR userdel)
Redacted events — sensor scrub
Sự kiện có token/password bị redact. Service nào leak nhiều nhất = ứng viên fix log format.
redacted:true | stats by (service, host) count() as leaks | sort by (leaks desc)
Firewall drops (nếu có iptables log)
Chỉ chảy khi iptables/nftables được cấu hình log rule.
service:kernel _msg:("IN=" AND "DROP")

Per-host drill down

Thay onehost-webhn072301 thành hostname client cần điều tra.

Toàn bộ log 1 host (30 phút)
Timeline chi tiết của 1 máy. Đủ để tra tay quanh incident time.
host:onehost-webhn072301
Error của 1 host (1h)
Zoom vào error của 1 máy — bỏ nhiễu host khác.
host:onehost-webhn072301 severity:err
Service phân bố trên 1 host
Xem host đang chạy các service nào — sanity check sau khi onboard.
host:onehost-webhn072301 | stats by (service) count() as n | sort by (n desc)
Live tail 1 host
Dán query rồi nhấn nút Live trong VMUI để stream realtime.
host:onehost-webhn072301

Volume & noise — ai spam log

Top talker host
Host nào gửi nhiều log nhất. Spike bất thường = spam từ 1 máy.
* | stats by (host) count() as n | sort by (n desc) | limit 10
Top talker service
Service nào đổ log nhiều nhất. Service log DEBUG có thể là ứng viên tune.
* | stats by (service) count() as n | sort by (n desc) | limit 15
Log rate theo phút (histogram nhỏ)
Xem trend ingest — spike/drop bất thường không?
* | stats by (_time:1m) count() as n | sort by (_time)
Message dài bất thường (> 8 KB)
Stack trace nhiều KB / binary blob có thể lãng phí storage.
* | filter len(_msg):>8192 | stats by (service, host) count()

LLM cost & usage

Tổng chi phí 24h
Cộng dồn cost mọi request qua LiteLLM. Đối chiếu với budget.
service:litellm_cost | stats sum(response_cost) as total_usd
Chi phí theo model
Model nào đang ăn nhiều $ — quyết định tune fallback/downgrade.
service:litellm_cost | stats by (model) sum(response_cost) as cost, count() as calls | sort by (cost desc)
Request chậm nhất (latency)
Top 20 request có duration cao nhất — spot timeout / thundering herd.
service:litellm_cost | sort by (duration_ms desc) | limit 20
Fallback đã trigger
Request nào phải fallback sang model dự phòng → primary provider có vấn đề.
service:litellm_cost fallback:true
Provider balance snapshot
Số dư gần nhất từng provider (DeepSeek, OpenAI, Anthropic). Cron poll mỗi 15m.
service:provider_cost
Token consumption theo model
Prompt vs completion tokens per model — tune prompt hoặc downgrade.
service:litellm_cost | stats by (model) sum(prompt_tokens) as prompt, sum(completion_tokens) as completion

System & kernel

OOM kill events
Kernel killed 1 process do hết RAM. Cần check container/process nào và tune limit.
_msg:("Out of memory" OR oom-killer OR "Killed process")
Disk I/O errors
SMART / block layer error — dấu hiệu disk sắp chết.
service:kernel _msg:("I/O error" OR "Buffer I/O" OR "medium error")
Filesystem read-only remount
Kernel remount FS thành RO khi phát hiện lỗi. App sẽ crash ngay sau đó.
_msg:("read-only" AND ("Remounting" OR "EXT4-fs"))
Systemd unit failure
Bất kỳ unit nào entered "failed" state trong 1h.
service:systemd _msg:("Failed" AND ".service")

Pipeline & probe health

Disk usage probe (logserver)
Log của cron probe check disk /opt/onelog/data. Alert khi > 75%.
service:logserver-disk-monitor
OpenWebUI SQLite size probe
Log của cron probe check webui.db. Alert khi > 500 MB (VACUUM cần thiết).
service:openwebui-db-monitor
Vector self-log (chạy trong docker)
Nhìn tổng hợp — Vector KHÔNG tự log ra VL để tránh loop. Chỉ có source_stream="vector-exec-probe".
source_stream:vector-exec-probe
Ingest rate — tất cả nguồn
Kiểm tra pipeline đang thực sự nhận log không.
* | stats count() as events, count_uniq(host) as hosts

severity:err OR severity:critKết hợp condition
